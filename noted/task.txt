user register 
  -> not exist
    -> add to database, set is validate to false
    -> create link, berisi hash dan id user 
    -> create email berisi link tersebut 
  -> cek if email exists
    -> cek if email validate 
      -> return false
    -> if email not validate
      -> delete user
      -> add to database, set validate to false 
      -> create link, berisi hash dan id user 
      -> create email berisi link tersebut

email verify
-> cek is hash exist in database
  -> cek apakah sudah expired 
    -> jika sudah 
      tampilkan ke halaman bahwa url sudah invalid 
    -> jika belum
      tambah data ke database 
      tampilkan ke halaman bahwa akun selesai diregistrasi silahkan login

api register otp
1. User register 
  cek apakah ada user dengan email yang sama
    jika tidak
    buat user tersebut 
    lalu kirimkan email berisi kode otp dan expired time
    return success,

    jika iya 
    cek apakah dia sudah divalidasi 
      jika sudah 
      tolak 

      jika belum 
      hapus user tersebut, 
      lalu buat user tersebut 
      lalu kirimkan email berisi kode otp dan expired time
      return success

api validate otp
2. validasi otp  
  cek apakah otp tersebut tersedia didatabase 
    jika otp tidak ada 
    return kode otp salah 

    jika otp tersedia, 
    cek apakah otp tersebut expired 
      jika iya, 
      hapus otp
      return kode otp expired silahkan registrasi kembali 
      frontend pindah halaman ke registrasi
     
      jika belum, 
      cek apakah otp tersebut benar milik user 
        jika benar 
        frontend redirect ke halaman login

        jika salah 
        return kode otp salah
